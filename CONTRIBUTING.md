# Contributing to the Ce Protocol DevNode-Expressive Layer

Contributions are welcome when they improve clarity, correctness, or applicability of the Ce protocol.

---

## Scope of Contributions

Acceptable contributions include:

* Clarifications to definitions
* Structural improvements
* Diagrams and explanatory material
* Reference architectures or experimental implementations

The protocol prioritizes **precision over expansion**.

---

## Contribution Principles

* Contributions are evaluated based on coherence with existing protocol definitions
* No identity-based authority is recognized
* Agreement is required for protocol-level changes

---

## Submitting Changes

1. Fork the repository
2. Create a clearly named branch
3. Submit a pull request with a concise explanation

Protocol changes may be discussed before acceptance.
