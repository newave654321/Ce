# JavaScript Simulations — Ce Protocol

## Purpose

The `software/simulations/js/` folder contains **browser- and Node.js-compatible simulations** of Ce behavior.  
These simulations emphasize **accessibility**, **visualization**, and **low-barrier participation**, including browser-based nodes.

JavaScript simulations are **non-normative** and intended for:
- Demonstration
- Education
- Rapid experimentation
- Browser-based participation

---

## Scope

This folder may include:

- Multi-node simulations
- Recall propagation tests
- Coherence visualization
- Failure and healing experiments
- Browser-compatible demos

JavaScript simulations focus on **interaction and visibility**, not performance or production deployment.

---

## Design Principles

- **Identity-free**  
  Nodes do not require persistent identifiers.

- **Event-driven**  
  Simulations use events, callbacks, or message passing rather than centralized control.

- **Browser-compatible**  
  Code should run in:
  - Modern browsers
  - Node.js (when possible)

- **Minimal dependencies**  
  Avoid heavy frameworks unless clearly justified.
