# Ce Protocol Specification

This directory contains the **core specification** of the Ce protocol.

Each document defines a specific aspect of the system using precise, minimal language intended for implementation, analysis, and reference.

---

## How to Read This Protocol

The protocol is organized by function, not chronology.

Recommended reading order:
1. `definitions.md`
2. `principles.md`
3. `coherence.md`
4. `agreement.md`
5. `recall.md`
6. `identity-vs-identifier.md`

Each file is self-contained but interdependent.

---

## Scope

These documents define:
- Terms and constraints
- Structural relationships
- Recognition and agreement mechanisms

They do **not** attempt to persuade or narrate.

---

## Relationship to the Books

The books provide:
- Motivation
- Extended examples
- Philosophical grounding

The protocol remains valid without them.

Book references are provided in `book-references/`.

---

## Diagrams

Visual representations of the protocol are stored in `diagrams/`.

Diagrams are explanatory, not normative.
